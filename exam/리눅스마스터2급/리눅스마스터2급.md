# 리눅스 마스터 2급 정리

## 1. 시그널

| 번호 | 시그널     | 설명                               |
| -: | ------- | -------------------------------- |
|  1 | SIGHUP  | 터미널과 연결이 끊어졌을 때 발생               |
|  2 | SIGINT  | 키보드로부터 오는 인터럽트 시그널               |
|  3 | SIGQUIT | 키보드로부터 오는 실행 중지 시그널              |
|  6 | SIGABRT | 비정상 종료                           |
|  9 | SIGKILL | 강제 종료                            |
| 14 | SIGALRM | 알람에 의해 발생                        |
| 15 | SIGTERM | `kill` 시스템 호출 시 발생               |
| 18 | SIGCONT | 중지된 프로세스 재실행                     |
| 19 | SIGSTOP | `SIGCONT` 시그널을 받을 때까지 프로세스 중지    |
| 20 | SIGTSTP | 실행 정지 후 다시 실행을 계속하기 위해 대기시키는 시그널 |

## 6. 파일 권한 및 소유권

### `ls -l` 출력 예시

```bash
-rw-r--r-- 1 root root 45 10월 11 13:44 linux.txt
```

| 항목             | 의미        |
| -------------- | --------- |
| `-`            | 파일 유형     |
| `rw-r--r--`    | 허가권       |
| `1`            | 링크 수      |
| `root`         | 소유자명      |
| `root`         | 그룹명       |
| `45`           | 파일 크기     |
| `10월 11 13:44` | 마지막 변경 시간 |
| `linux.txt`    | 파일명       |

### 파일 유형

| 문자  | 의미        |
| --- | --------- |
| `-` | 일반 파일     |
| `d` | 디렉터리      |
| `l` | 심볼릭 링크 파일 |
| `c` | 문자 장치 파일  |
| `b` | 블록 장치 파일  |
| `s` | 소켓        |
| `p` | 파이프       |

### 소유권 변경

| 명령어     | 설명                            |
| ------- | ----------------------------- |
| `chown` | 파일이나 디렉터리의 사용자 소유권과 그룹 소유권 변경 |
| `chgrp` | 파일이나 디렉터리의 그룹 소유권 변경          |

```bash
chown test linux.txt
chown test.test linux.txt
chgrp test linux.txt
```

### 허가권

| 권한     | 숫자 |
| ------ | -: |
| 읽기 `r` |  4 |
| 쓰기 `w` |  2 |
| 실행 `x` |  1 |

| 대상  | 의미     |
| --- | ------ |
| `u` | 소유자    |
| `g` | 그룹     |
| `o` | 기타 사용자 |
| `a` | 모든 사용자 |

| 연산자 | 의미     |
| --- | ------ |
| `+` | 허가권 부여 |
| `-` | 허가권 제거 |
| `=` | 허가권 지정 |

### chmod 예시

```bash
chmod 755 linux.txt
# -rwxr-xr-x

chmod u-x,g+w,o-x linux.txt
# -rw-rwxr--

chmod a=r gnu
# 모든 사용자에게 읽기 권한만 부여
```

### umask

`umask`는 새로 생성되는 파일이나 디렉터리의 기본 허가권을 지정하는 명령어이다.

| 구분   | 기본 권한 |
| ---- | ----- |
| 파일   | 666   |
| 디렉터리 | 777   |

예시:

```bash
umask 022
```

| 구분   | 계산        | 결과  |
| ---- | --------- | --- |
| 파일   | 666 - 022 | 644 |
| 디렉터리 | 777 - 022 | 755 |

### 특수 허가권

| 권한         | 문자  |   숫자 | 설명                        |
| ---------- | --- | ---: | ------------------------- |
| SetUID     | `s` | 4000 | 파일 실행 시 파일 소유자의 권한으로 실행   |
| SetGID     | `s` | 2000 | 파일 실행 시 파일 소유 그룹의 권한으로 실행 |
| Sticky Bit | `t` | 1000 | 디렉터리 내 파일 삭제 권한 제한        |

* 소문자 `s`, `t`: 실행 권한이 있는 상태
* 대문자 `S`, `T`: 실행 권한이 없는 상태

```bash
chmod 4744 linux.txt
# -rwsr--r--

chmod u-x linux.txt
# -rwSr--r--

chmod 6754 linux.txt
# -rwsr-sr--

chmod ug-s linux.txt

chmod 1754 gnu
# drwxr-xr-T
```

## 7. 디스크 쿼터

디스크 쿼터는 파일 시스템별로 사용자나 그룹의 사용량을 제한할 때 사용한다.

| 명령어          | 설명                         |
| ------------ | -------------------------- |
| `quotaon`    | 쿼터 활성화                     |
| `quotaoff`   | 쿼터 사용 중지                   |
| `quotacheck` | 파일 시스템 점검 및 쿼터 설정/기록 파일 생성 |
| `edquota`    | 사용자와 그룹에 대해 쿼터 설정 및 변경     |
| `setquota`   | 터미널에서 직접 사용자나 그룹에 쿼터 적용    |
| `repquota`   | 사용자나 그룹에 적용된 쿼터 요약 출력      |
| `xfs_quota`  | XFS 파일 시스템에서 쿼터 설정         |

## 10. 프로세스와 작업 관리

### foreground와 background

| 구분         | 설명                                     |
| ---------- | -------------------------------------- |
| foreground | 명령 실행 결과가 나올 때까지 기다리는 일반적인 실행 방식       |
| background | 프로세스 실행 중에도 다른 명령을 실행할 수 있는 방식, `&` 사용 |

```bash
command &
```

### fork와 exec

| 구분       | 설명                                       |
| -------- | ---------------------------------------- |
| `fork()` | 기존 프로세스를 복제하여 새로운 프로세스 생성                |
| `exec()` | 새로운 프로세스를 생성하지 않고 기존 프로세스가 다른 프로그램으로 대체됨 |

### 데몬

| 구분         | 설명                       |
| ---------- | ------------------------ |
| Daemon     | 시스템 부팅 시 실행되는 백그라운드 프로세스 |
| standalone | 메모리에 상주하다가 요청 시 바로 서비스   |
| inetd      | 필요한 서비스만 메모리에 적재하여 실행    |

### 프로세스 명령어

| 명령어       | 설명                            |
| --------- | ----------------------------- |
| `ps`      | 현재 실행 중인 프로세스 상태 출력           |
| `pstree`  | 실행 중인 프로세스를 트리 구조로 출력         |
| `top`     | 프로세스 상태를 실시간 확인               |
| `nohup`   | 터미널 종료 후에도 프로세스를 계속 실행        |
| `jobs`    | 백그라운드 작업 확인                   |
| `bg`      | foreground 작업을 background로 전환 |
| `fg`      | background 작업을 foreground로 전환 |
| `nice`    | 프로그램 실행 시 우선순위 지정             |
| `renice`  | 실행 중인 프로세스의 우선순위 변경           |
| `kill`    | 프로세스에 특정 Signal 전송            |
| `killall` | 프로세스 이름으로 종료                  |

### ps 옵션

| 옵션   | 설명                             |
| ---- | ------------------------------ |
| `-a` | 로그인 셸과 터미널에 종속되지 않은 모든 프로세스 출력 |
| `-e` | 커널 프로세스를 제외한 모든 프로세스 출력        |
| `-f` | full 포맷 출력                     |
| `-l` | 자세한 프로세스 목록 출력                 |
| `-o` | 출력 포맷 지정                       |
| `-p` | 특정 PID 지정                      |
| `-r` | 현재 실행 중인 프로세스 출력               |
| `-u` | 특정 사용자의 프로세스 확인                |
| `-x` | 접속 터미널뿐 아니라 사용 중인 모든 프로세스 출력   |

### nice 값

* 범위: `-20` ~ `19`
* 수치가 낮을수록 우선순위가 높다.
* 일반 사용자는 nice 값을 증가만 할 수 있다.

## 12. RAID

| -------- | ------------------------------------------- |
| RAID 0   | 스트라이핑 사용, 입출력 빠름, 하나의 디스크 오류 시 모든 데이터 손실 가능 |

## 18. 네트워크 토폴로지

| 구조  | 설명                                 |
| --- | ---------------------------------- |
| 스타형 | 중앙 컴퓨터에 여러 대의 컴퓨터가 허브나 스위치로 연결된 구조 |
| 버스형 | 하나의 통신 회선에 여러 대의 컴퓨터를 연결한 구조       |
| 링형  | 각 노드가 인접한 노드와 연결되어 원형을 이룬 구조       |
| 트리형 | 버스형과 스타형 구조의 결합 구조                 |
| 망형  | 모든 노드가 서로 일대일로 연결된 구조              |

### 스타형

* 중앙 집중식 구조로 네트워크 확장이 용이하다.
* 고속의 대규모 네트워크에 적합하다.
* 중앙 컴퓨터 장애 시 전체 네트워크에 영향을 준다.

### 버스형

* 설치 비용이 저렴하다.
* 단말 장치 추가 및 제거가 용이하다.
* 컴퓨터 수 증가 시 병목 현상이 발생할 수 있다.
* 장애가 발생한 노드의 위치 파악이 어렵다.

### 링형

* 토큰 패싱 방식으로 데이터를 전송한다.
* 데이터 전송 시 충돌이 발생하지 않는다.
* 노드 수가 증가해도 네트워크 성능 저하가 적다.
* 하나의 노드 장애가 전체 네트워크에 영향을 줄 수 있다.
* 노드 추가 및 제거가 쉽지 않다.

### 트리형

* 백본 네트워크에 적합하다.
* 분기 장치를 사용하여 링크를 추가할 수 있다.
* 트래픽 증가 시 병목 현상이 발생할 수 있다.

### 망형

* 다량의 데이터 전송에 적합하다.
* 장애 발생 시 다른 노드에 영향이 적다.
* 우회 경로가 있어 신뢰성이 높다.
* 회선 구축 비용이 많이 든다.

## 19. 네트워크 종류

| 종류         | 설명                                            |
| ---------- | --------------------------------------------- |
| Ethernet   | IEEE 표준, CSMA/CD 사용                           |
| Token Ring | 데이터가 한쪽 방향으로 전달, 확장성이 좋지 않음                   |
| FDDI       | ANSI 표준, LAN 고속 스테이션 연결 및 백본망 연결에 사용, 이중 링 구조 |
| MAN        | 도시권 통신망, 여러 LAN을 연결하여 백본 라인 구성                |
| WAN        | 국가나 대륙 같은 넓은 지역을 연결, 전송 오류율이 높음               |
| SAN        | 저장 장치를 위해 설계된 고속 네트워크                         |

## 20. 네트워크 접근 방식

| 방식            | 설명                            |
| ------------- | ----------------------------- |
| CSMA/CD       | 전송로의 사용 여부를 확인하고 데이터를 전송하는 방식 |
| Token Passing | 토큰의 흐름에 의해 전송 순서가 결정되는 방식     |

Token Passing은 Free Token과 Busy Token을 이용하여 전송 매체 접근을 제어한다.

## 21. 네트워크 프로토콜

| 프로토콜 | 설명                                     |
| ---- | -------------------------------------- |

* 비연결성, 비신뢰성 서비스 제공

* 최대 전송 단위를 초과하는 패킷의 단편화 및 재조립 수행

### ARP 특징

* 요청은 브로드캐스트
* 응답은 유니캐스트

## 30. 클러스터링

클러스터링은 여러 개의 시스템을 네트워크로 연결하여 하나의 큰 시스템처럼 동작하도록 만드는 기술이다.

| 종류        | 설명                                         |
| --------- | ------------------------------------------ |
| 고계산용 클러스터 | HPC Cluster, 베어울프 클러스터, 과학 계산용             |
| 부하분산 클러스터 | LVS Cluster, 대규모 서비스 제공, 여러 서버가 부하분산기에 연결  |
| 고가용성 클러스터 | HA Cluster, 연속적인 서비스 제공, 금융권과 데이터 센터 등에 사용 |

## 32. 클라우드 서비스 모델

| 모델   | 설명                                 |
| ---- | ---------------------------------- |
| IaaS | 서버, 스토리지, 네트워크 등 하드웨어 자원을 임대하는 서비스 |
| SaaS | 클라우드 환경에서 실행되는 응용 프로그램을 서비스 형태로 제공 |
| PaaS | 소프트웨어 개발을 위한 플랫폼을 제공하는 서비스         |

### IaaS

* 사용자는 하드웨어를 보유하지 않고 서버, 스토리지, 네트워크 등의 자원을 사용할 수 있다.
* 필요에 따라 자원을 추가하거나 삭제할 수 있다.

### SaaS

* 필요한 기능을 필요한 만큼만 임대하는 방식이다.
* 클라우드 환경에서 실행되는 응용 프로그램을 서비스 형태로 제공한다.

### PaaS

* 애플리케이션 개발 환경을 제공한다.
* 환경 구축의 어려움을 줄이고 단기간에 서비스를 개발할 수 있다.
